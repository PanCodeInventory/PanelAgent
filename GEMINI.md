# GEMINI.md

## Project Overview

This project is a Python-based "smart flow cytometry panel generator" that leverages a locally hosted Large Language Model (LLM) to automate the design of multi-color flow cytometry panels. The application is built with a Streamlit-based user interface, allowing users to input a list of markers and receive an optimized panel design based on the laboratory's actual antibody inventory.

The project follows a "Sandwich" architecture:
1.  **Data Preprocessing (Python):** Loads and preprocesses antibody inventory data from a CSV file. It normalizes marker names and maps fluorochrome names to standardized system codes.
2.  **Core Decision (LLM):** A locally hosted LLM (via LM Studio) receives the preprocessed data and the user's desired markers. It suggests an initial panel based on expert rules (e.g., brightness matching).
3.  **Validation and Refinement (Python):** The application's core logic validates the LLM's suggestion, scores it based on physical rules (e.g., brightness constraints), and can iteratively critique and refine the panel by communicating with the LLM. It then presents the final, validated panel to the user, along with a design rationale and gating strategy generated by the LLM.

## Building and Running

### Prerequisites

*   Python 3.9+
*   LM Studio running with a loaded GGUF model.
*   The required Python packages: `streamlit`, `pandas`, `openai`

### Key Commands

1.  **Install dependencies:**
    ```bash
    pip install pandas streamlit openai
    ```

2.  **Run the application:**
    ```bash
    streamlit run streamlit_app.py
    ```
    The application will be available at a local URL in your web browser.

## Development Conventions

*   **Modular Architecture:** The code is organized into distinct modules with clear responsibilities:
    *   `streamlit_app.py`: Handles the user interface and application flow.
    *   `panel_generator.py`: Contains the core logic for panel generation, scoring, and refinement.
    *   `llm_api_client.py`: Manages communication with the local LLM.
    *   `data_preprocessing.py`: Responsible for loading and preparing all necessary data.
*   **Configuration Files:** The application uses JSON files for configuration:
    *   `channel_mapping.json`: Standardizes fluorochrome names.
    *   `fluorochrome_brightness.json`: Provides brightness scores for fluorochromes, which is a key parameter for the panel optimization logic.
*   **LLM Interaction:** The system communicates with the LLM using the `openai` Python library, configured to point to a local LM Studio instance. Prompts are engineered to be specific and request JSON output for reliable parsing.
*   **Error Handling:** The code includes error handling for LLM communication and JSON parsing, with retry mechanisms suggested in the design documentation.
*   **Data-driven:** The panel generation is driven by the contents of `流式抗体库-20250625小鼠.csv`, making it adaptable to different antibody inventories.
